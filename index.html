Вставить чек-лист из Jira

Скопируйте содержимое комментария из режима Jira «Текст». Таблицы, разделы, статусы и форматирование будут преобразованы в редактируемый отчёт.

Вставьте прямую ссылку на комментарий Jira. Подключение берётся из настроек Jira.

Перед импортом текущий отчёт будет сохранён в истории.

Настройки

Интеграции

Токен, пароль или cookie хранятся только в памяти открытой вкладки и исчезают после перезагрузки. Остальные настройки сохраняются локально в браузере.

Можно вставить curl с Bearer, Basic, -u user:password или Cookie. Секрет из curl будет применён только в текущей вкладке.

Соединение ещё не проверялось.
Токены дисков используются только сервером приложения для загрузки файлов. В браузере сохраняются только признаки активности и путь/ID папки.

Создайте OAuth-токен для Яндекс.Диска с доступом к REST API Диска. Укажите папку, куда будут попадать изображения.

Получите access token с областью доступа Drive File или Drive. ID папки берётся из адреса открытой папки Google Drive.

Настройки файлового хранилища ещё не сохранялись.
Облачное хранилище позволяет открывать чек-листы в разных браузерах при одинаковом ключе пространства. Если опция выключена, данные остаются только в этом браузере.

Одинаковый ключ пространства позволяет использовать одну облачную историю в разных браузерах. Если поле пустое, используется этот браузер.

Чек-листы сохраняются только локально в этом браузере.
